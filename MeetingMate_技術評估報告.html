MeetingMate

語音轉文字與即時翻譯技術評估報告
2026年3月30日
需求：即時語音轉文字（含英譯中）+ 即時重點摘要 + 錄音檔會議記錄與翻譯

一、評估背景與目標

MeetingMate 目前已實現離線錄音檔的語音轉文字（WhisperX + pyannote speaker diarization）、LLM 會議摘要、與 Google Drive 備份功能。現在希望擴展兩大功能：

  • 功能 A：即時語音轉文字（Real-time STT）——含英文即時翻譯成中文，以及即時重點摘要
  • 功能 B：錄音檔翻譯（Batch Translation）——將英文錄音檔轉錄後翻譯成中文

二、技術方案總覽

2.1 Microsoft VibeVoice-ASR

2026年1月21日由微軟開源，是一個 9B 參數的統一語音辨識模型，可在單次推論中處理長達 60 分鐘的音訊。

特性說明
模型大小9B 參數，需 GPU（建議 24GB+ VRAM）
核心功能ASR + Speaker Diarization + Timestamping 三合一
單次推論同時輸出「誰說的」「什麼時候」「說了什麼」
語言支援50+ 語言，原生支援中英文 code-switching
中文表現在 AISHELL-4、AliMeeting 等中文 benchmark 上測試，超越 Gemini-2.5-Pro
即時性批次處理為主（Batch），非即時串流設計
可透過 vLLM 優化推論速度
Hotwords支援自定義熱詞（人名、專有名詞）提升辨識率
翻譯功能無內建翻譯，需外掛 LLM 處理

2.2 WhisperX（目前 MeetingMate 使用）

WhisperX = faster-whisper + wav2vec2 alignment + pyannote diarization，是目前 MeetingMate 的轉錄引擎。

特性說明
模型大小large-v3 約 3GB，CPU 可運行（int8）
核心功能ASR + 字級時間戳 + Speaker Diarization（三個模型串聯）
DER 約 8%，比單獨 pyannote 更低
語言支援99 種語言（Whisper 原生支援）
即時性批次處理（Batch only），不支援串流
翻譯功能Whisper 內建 X→英文翻譯，但不支援英→中文

四、建議架構設計

根據需求分析，建議將 MeetingMate 擴展為「雙引擎」架構：

4.1 功能 A：即時語音轉文字 + 即時翻譯 + 即時重點

注意：VibeVoice-ASR 與 WhisperX 都不支援即時串流，即時功能需要不同的技術組合。
層建議方案
即時 STT 方案 A：Azure Speech SDK（雲端，中英文最佳，付費）
方案 B：faster-whisper streaming mode（離線，將音訊切割成小塊遞增轉錄）
方案 C：Vosk（離線即時，精確度較低但延遲極低）
即時英→中翻譯 LLM API 即時翻譯（Streaming）：
每當累積一句完整的英文，透過 GPT-4o / Claude / Gemini API 即時翻譯成中文，以 SSE 推送到前端
即時重點摘要 LLM 滾動摘要：
每 2–3 分鐘將累積的轉錄內容傳給 LLM，產生「即時重點」顯示在側欄
會議結束後再用完整轉錄稿產生正式摘要

4.2 功能 B：錄音檔會議記錄 + 翻譯（已有基礎）

層建議方案

英→中翻譯 新增功能：轉錄完成後，用 LLM 將英文轉錄稿翻譯成中文
可作為「翻譯」按鈕加在會議詳細頁

五、實作優先級建議

根據技術複雜度、實用性和成本考量，建議以下實作順序：

Phase 1：錄音檔翻譯功能（最簡單，1–2 天）

  1. 在現有會議詳細頁新增「翻譯」按鈕，轉錄完成後可將英文轉錄稿透過 LLM 翻譯成中文
  2. 使用現有 LLM API（GPT-4o / Claude / Gemini），無需新引擎
  3. 難度低，只需新增一個 API endpoint + 前端按鈕

Phase 2：即時語音轉文字（中等難度，3–5 天）

  1. 新增「即時轉錄」頁面，透過瀏覽器麥克風擷取音訊
  2. 建議採用 faster-whisper 切塊遞增轉錄（每 5 秒處理一次）——離線可用、免費
  3. 或採用 Azure Speech SDK（雲端）——中英文混合最佳，但需付費

Phase 3：即時英→中翻譯 + 即時重點（較高難度，3–5 天）

  1. 即時轉錄的英文字幕透過 LLM Streaming API 即時翻譯成中文
  2. 每 2–3 分鐘用 LLM 產生即時重點，顯示在側欄
  3. 需要 WebSocket / SSE 實現前後端即時通訊

Phase 4（選配）：升級為 VibeVoice-ASR

  1. 若未來有 GPU 伺服器，可將離線轉錄引擎升級為 VibeVoice-ASR
  2. 優勢：更高的中文精確度、內建 diarization、支援 hotwords
  3. 前提：需要 24GB+ GPU VRAM（如 RTX 4090 或雲端 GPU）

七、結論

VibeVoice-ASR 是目前最強的開源語音辨識模型，但它並非即時串流引擎，而是批次處理模型，且硬體需求高（24GB+ GPU）。它沒有內建翻譯功能。

對於 MeetingMate 的即時需求，最務實的做法是：

  • 離線轉錄：繼續使用 WhisperX（已有），CPU 可運行，夠用
  • 即時轉錄：用 faster-whisper 切塊遞增 或 Azure Speech SDK
  • 英→中翻譯：統一用 LLM API 處理（即時和離線都適用）
  • 即時重點：LLM 滾動摘要（每 2–3 分鐘更新一次）
  • VibeVoice-ASR：留待未來有 GPU 時再升級，作為 Phase 4 選配
建議從 Phase 1（錄音檔翻譯）開始實作，這是最快能看到效果的功能，且技術難度最低。
